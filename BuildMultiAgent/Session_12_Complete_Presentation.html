When Do You Need Multi-Agent Systems?

Decision Framework

Use Single Agent When:

  • Task is well-defined and narrow
  • One domain of expertise
  • Simple tool usage
  • Linear workflow

Examples:

  • Customer FAQ bot
  • Document summarizer
  • Code reviewer

Use Multi-Agent System When:

  • Multiple distinct steps with different skills
  • Need parallelization for speed
  • Different levels of expertise required
  • Complex decision trees

Examples:

  • College admissions (query handling + document processing + eligibility check)
  • E-commerce (product search + recommendation + inventory + pricing)
  • Research assistant (literature search + summarization + comparison + citation)

Problem Analysis Framework

5-Step Process Before You Code Step 1: Understand the Problem
  • What is the end-to-end workflow?
  • Who are the users?
  • What are the pain points?
  • What does success look like?
Step 2: Identify Sub-Tasks
  • Break down the problem into discrete steps
  • Each step = potential agent
Step 3: Define Agent Roles
  • What expertise does each step need?
  • What tools are required?
  • What inputs/outputs?
Step 4: Design Communication Flow
  • Which agents need to talk to each other?
  • Sequential or parallel execution?
  • How is data passed between agents?
Step 5: Identify Success Metrics
  • How do you measure if it works?
  • Performance requirements?
  • Quality benchmarks?

College Admission Management System - Introduction

Real Problem:

  • University receives 5,000 applications during peak season
  • Staff overwhelmed answering repetitive questions (100+ queries/day)
  • Document verification takes 2-3 weeks per application
  • Inconsistent eligibility evaluation across reviewers
  • Poor student communication leads to incomplete applications
Our Solution: Multi-Agent System with File Processing Agent 1: Query Handler (Q&A Bot)
  • What it does: Answers student questions 24/7 using FAQ database and program information
  • Example: "What is the application deadline?" â†’ "Application deadline is November 30th, 2025"
  • Tools: FAQ_Search, Program_Info (CS, EE, ME programs)
Agent 2: Document Processor (Data Extractor)
  • What it does: Reads and extracts structured data from unstructured text files
  • Processes: transcript.txt (GPA, subjects, graduation year), recommendation.txt (3 key points), essay.txt (themes, writing quality, authenticity scores)

  • Example Output: JSON {"gpa": "3.9", "subjects": ["Mathematics", "Physics"], "graduation_year": "2025"}

  • What it does: Analyzes extracted data against admission criteria and calculates scores

class ProgressMonitor:


    def __init__(self, llm):


        self.llm = llm


        self.student_progress = {}



    def track_completion(self, student_id, completed_item):


        """Track what student completed"""


        if student_id not in self.student_progress:


            self.student_progress[student_id] = []



        self.student_progress[student_id].append(completed_item)



    def evaluate_progress(self, student_id, original_plan):


        """Evaluate if student is on track"""


        progress = self.student_progress.get(student_id, [])



        prompt = f"""Evaluate student progress:



        Original Plan: {json.dumps(original_plan, indent=2)}



        Completed So Far: {json.dumps(progress, indent=2)}



        Determine:


    

  1. Is student on track? (Yes/No/Behind/Ahead)
    2. 

  2. Should we adjust the plan?
    3. 

  3. What encouragement/feedback to provide?
    4. 

  4. Any red flags?
    5. 




        Return as JSON.


        """



        evaluation = self.llm.invoke(prompt)


        return evaluation



    def adapt_plan(self, current_plan, evaluation):


        """Adapt plan based on progress"""


        if "behind" in evaluation.lower():


            # Simplify or extend timeline


            return self.simplify_plan(current_plan)


        elif "ahead" in evaluation.lower():


            # Add more advanced content


            return self.accelerate_plan(current_plan)


        else:


            return current_plan
